Primera forma normal

Área de producción

ESCENOGRAFÍA ( ID, ID_ESPACIO(FK), COD_ACTUACIÓN(FK) )

METERIAL_ESCENOGRAFÍA ( ID_ESCENOGRAFIA(PK)(FK), ID_MATERIAL(PK)(FK) )

ESPACIOS ( ID_ESPACIO(PK), TIPO, LOCALIZACIÓN, EMPLEADO_ENCARGADO, CAPACIDAD )

MATERIAL ( ID_MATERIAL(PK), TIPO, PRECIO, TIEMPO_DISPONIBLE, PROVEEDOR, CANTIDAD, ALMACENAJE, DESCRIPCION )

MATERIAL_ESPACIO ( ID_MATERIAL(PK)(FK), ID_ESPACIO(PK)(FK) )

CATERING ( ID_CATERING(PK), EMPRESA_ENCARGADA, PRESUPUESTO )

PUESTO_CATERING ( ID_CATERING(PK)(FK), NUM_PUESTO, LUGAR_ASIGNADO, SERVICIO, SERVICIO_GRATUITO )

CATERING_ESPACIO ( ID_CATERING(PK)(FK), ID_ESPACIO(PK)(FK) )

La primera forma normal considera que cada atributo debe ser atómico(contener un solo valor) para evitar la pérdida de datos. Para conseguirlo, en el área de producción debemos separar la tabla de CATERING en dos; una con los datos relacionados con los datos relacionados con la ubicación (el puesto), y otra con la empresa y el presupuesto (si no lo dividiesemos, la tabla tendría multiples valores en el atributo lugares)

Área artística

ARTISTA ( ID_ARTISTA(PK)(FK), ID_ESPACIO(FK), NOMBRE_ARTISTICO, NOMBRE_LEGAL, DNI/PASPORTE, FECHA_NACIMIENTO, CAMPO_ARTISTICO, TELEFONO, SUELDO )

CARTEL ( ID_CARTEL(PK), WEB, PRECIO_ENTRADA )

ARTISTAS_CARTEL ( ID_CARTEL(PK)(FK), ID_ARTISTA(PK)(FK) )

PATROCINADORES_CARTEL ( ID_CARTEL(PK)(FK), PATROCINADORES )

AGENDA ( COD_ACTUACION(PK), TELONERO, FECHA, HORARIO )

AGENDA_ARTISTA ( ID_ARTISTA(PK)(FK), COD_ACTUACION(PK)(FK) )

AGENDA_CARTEL ( ID_CARTEL(PK)(FK), COD_ACTUACION(PK)(FK) )

INVITACIONES ( COD_INVITACION(PK), ID_ARTISTA(PK)(FK), NOMBRE, NUM_ACOMPAÑANTES, PERMISOS )

REDES_SOCIALES_ARTISTAS ( ID_CUENTA(PK), ID_ARTISTA(PK)(FK), PLATAFORMA, NOMBRE_CUENTA )

En el caso del área artística, el atributo de redes_sociales tiene múltiples valores(uno por plataforma), por lo que creamos una tabla con los datos de dichas cuentas. A su vez, los patrocinadores que aparecerán en los carteles también están en un atributo (que sería multivaluado), por lo que lo separamos en una tabla diferente con el cartel en el que aparece

Segunda forma normal

Área de producción

ESCENOGRAFÍA ( ID, ID_ESPACIO(FK), COD_ACTUACIÓN(FK) )

METERIAL_ESCENOGRAFÍA ( ID_ESCENOGRAFIA(PK)(FK), ID_MATERIAL(PK)(FK) )

ESPACIOS ( ID_ESPACIO(PK), TIPO, LOCALIZACIÓN, EMPLEADO_ENCARGADO, CAPACIDAD )

MATERIAL ( ID_MATERIAL(PK), TIEMPO_DISPONIBLE, CANTIDAD, ALMACENAJE )

MATERIAL_DATOS ( ID_MATERIAL(PK), TIPO, PRECIO, PROVEEDOR, DESCRIPCION )

MATERIAL_ESPACIO ( ID_MATERIAL(PK)(FK), ID_ESPACIO(PK)(FK) )

CATERING ( ID_CATERING(PK), EMPRESA_ENCARGADA, PRESUPUESTO )

PUESTO_CATERING ( ID_CATERING(PK)(FK), NUM_PUESTO, LUGAR_ASIGNADO, SERVICIO, SERVICIO_GRATUITO )

CATERING_ESPACIO ( ID_CATERING(PK)(FK), ID_ESPACIO(PK)(FK) )

Para la segunda forma normal, es necesario que esté en 1FN y además todo atributo no clave debe depender de toda la clave principal. En el área de producción, el tipo, descripción, precio y proveedor del material son independientes del tiempo que lo alquilen y dónde lo almacenen. El área dartística ya se encuentra en segunda forma normal

Tercera forma normal

Para la TERCERA forma normal, es necesario que esté en 2FN y además que cada atributo no clave dependa de la clave primaria de modo no transitivo (ningún atributo puede depender de otro no clave, y todos deben ser dependientes funcionales de la clave primaria. Todas las tablas de ambas áreas se encuentran en tercera forma normal
